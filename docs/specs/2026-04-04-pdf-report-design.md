# PDF分析レポート機能 設計書

## 概要

kabu-dexterの総合分析結果を、初心者にもわかりやすいビジュアルPDFレポートとして出力する機能。レーダーチャートと横棒グラフで8軸のスコアを可視化し、数値の意味づけ（業界比較+過去比較）をストーリー型で伝える。ラジ株ナビMCP v2（2026-04-04より）の80指標に対応。

## 背景と目的

### 課題
- 現在の総合分析出力は専門用語が多く、初心者には理解が困難
- 数値が並ぶだけで「結局どうすればいいのか」がわからない
- 情報量が多すぎて圧倒される

### 目的
- 投資初心者〜中級者が一目で銘柄の全体像を把握できるレポート
- 数値を「どう見ているか」（業界比較+過去比較）を常に添えて解釈を提供
- 保存・共有しやすいPDF形式で出力

### 将来展望
- ラジ株ナビ（Web版）への組み込み
- note有料プランでのレポート提供

## ターゲットユーザー

- A) 株式投資自体が初めて（PERやROEに馴染みがない）
- B) 投資経験はあるが分析ツールは使ったことがない
- C) 分析手法は知っているがツールの操作がハードル

全レベルをカバーする。

## 技術スタック

- **テンプレート**: HTML + Chart.js（CDN）
- **PDF化**: Playwright（既にプロジェクトの依存に含まれている）
- **ランタイム**: Bun（既存プロジェクトに合わせる）
- **理由**: HTML→PDF変換はデザイン自由度が高く、ユーザー環境にPythonを要求しない。Playwrightは`src/tools/browser/browser.ts`で既に使用されており、追加依存なしで実装可能

## デザイン方針

### デジタル庁ダッシュボードガイド準拠

参考: https://www.digital.go.jp/resources/dashboard-guidebook

以下の原則に従う:

#### 2大原則
1. **「知りたいことを知れる」** - シンプル、意味のある順列、強弱をつける
2. **「誤解を生まない」** - わかりやすく表記、データを定義、表現を歪曲しない、メタ情報を記載

#### カラーパレット

| トークン | HEX | 用途 |
|---------|-----|------|
| Blue 600 | `#3460FB` | キーカラー、基準値50以上のスコア |
| Blue 900 | `#0017C1` | ヘッダーテキスト |
| Blue 400 | `#7096F8` | SMA50ライン |
| Blue 200 | `#C5D7FB` | 出来高、薄い背景 |
| Blue 50 | `#D9E6FF` | セクションタイトル背景 |
| Red 600 | `#FE3939` | 基準値50以下のスコア |
| Solid Gray 400 | `#999999` | 基準値ライン |
| Solid Gray 200 | `#CCCCCC` | ボーダー |
| Standard BG | `#F8F8FB` | ページ背景 |

#### グラフルール
- グラフの色数は1〜5色に絞る
- コントラスト比3:1以上確保
- 棒グラフの原点は必ず0
- データ点にマーカーを付ける
- 凡例はグラフの近傍に配置
- 色のみで分類を識別しない（形状でも区別）
- タイトルにデータ種別を明記
- メタ情報（データソース・更新日）を記載

#### レイアウト
- 左上→右下の視線の流れ
- 全体→詳細の階層構造
- グリッドベース

#### レーダーチャートについて
デジタル庁ガイドの推奨グラフ（折れ線・棒・面・円）にレーダーチャートは含まれない。本レポートでは直感的な理解のためにレーダーチャートを採用しつつ、正確な比較用に横棒グラフも併記する。カラーパレット・レイアウト原則・アクセシビリティはガイドに準拠する。

## レポート構成

1つの連続したレポート（強制改ページなし）。PDF化時に自然な位置で改ページされる。

```
┌─────────────────────────────────────────────┐
│ ヘッダー: 銘柄情報                            │
├──────────────────────┬──────────────────────┤
│ レーダーチャート       │ 総合コメント          │
│ （8軸、自社+基準値50） │ （ストーリー型）      │
├──────────────────────┴──────────────────────┤
│ 8軸スコアカード（横一列）                      │
├─────────────── 区切り線 ────────────────────┤
│ 各軸の詳細（カード型、2カラムグリッド）         │
│ + 各軸の詳細（カード型、2カラムグリッド）         │
├─────────────── 区切り線 ────────────────────┤
│ 株価チャート（折れ線+出来高+SMA）              │
├─────────────────────────────────────────────┤
│ リスク要因                                   │
├─────────────────────────────────────────────┤
│ データソース・分析手法補足                     │
├─────────────────────────────────────────────┤
│ フッター（免責事項・更新日）                   │
└─────────────────────────────────────────────┘
```

### ヘッダー
- 銘柄名（大きく）、銘柄コード、セクター
- 現在株価、時価総額、分析日

### レーダーチャート
- 8軸、0〜100スケール
- 自社スコア: Blue 600（半透明塗り、丸マーカー）
- 基準値（50）: Solid Gray 400（破線、三角マーカー）
- 凡例はチャート近傍に配置

### 総合コメント
- ストーリー型で3〜5行
- 「この会社は〜が強いが、〜に陰りがあり、〜」の形式
- 数値を根拠として織り込む

### 8軸スコアカード
- 横一列のカード型
- 各カード: 軸名 + スコア数値 + ミニバー
- 基準値50以上→Blue、以下→Red で色分け

### 各軸の詳細
- 2カラムグリッドでカード表示
- 各カード: 軸名 + スコア(/100) + プログレスバー（基準値50マーカー付き）+ 根拠テキスト
- 根拠は「数値 → 比較対象 → 解釈」の3点セット
  - 絶対基準との比較（「PER 7.3倍は一般的に割安水準」）
  - 過去の自分との比較（縦の比較）

### 株価チャート
- 直近12週の日足（JQuants無料プランの範囲）
- 折れ線: 終値（Blue 600、マーカー付き）
- 破線: SMA50（Blue 400）
- 棒グラフ: 出来高（Blue 200、第2軸）
- SMA200: 無料プランでは算出不可。Light以上で表示（注記付き）

### リスク要因
- 箇条書き2〜3項目
- 分析結果から特に懸念される点を抽出

### フッター
- データソース: ラジ株ナビ / JQuants / TDnet
- 更新日
- 免責事項（投資助言ではない旨）

## 8軸スコア定義

全軸0〜100。50を「普通」の基準とする。全て絶対基準でスコア化する（業界平均の実データは使用しない）。

各軸は**主指標**（ベーススコアを決定）と**補助指標**（加減点）で構成する。主指標でベーススコア（0〜100）を算出し、補助指標で加重平均する。最終値は0〜100にクランプ。

### 1. 割安度（Valuation）

| 指標 | 役割 | 変換ルール |
|------|------|----------|
| DCF乖離率 | 主指標（50%） | +30%以上割安→90、±0%→50、-30%以上割高→10（線形補間） |
| EV/EBITDA | 補助（20%） | 5倍以下→90、15倍以上→10（絶対基準、線形補間） |
| PER | 補助（15%） | 8倍以下→90、25倍以上→10（絶対基準、線形補間） |
| PBR | 補助（10%） | 0.5以下→80、1.0以下→65、2.0以下→45、2.0超→25 |
| ネットネット判定 | 補助（5%） | NCAV/時価総額>1.5→100、1.0以上→85、0.67以上→65 |

### 2. 稼ぐ力（Profitability）

- 「PDFレポートを出して」
- 「レポートにして」
- 「PDF出して」

```
comprehensive-analysis（総合分析）
  └─ Phase 7 完了後
```

- `output/` ディレクトリは `generate.ts` で自動作成する。`.gitignore` に追加する

## ISQスコアから8軸スコアへの変換

総合分析の各フェーズはISQスコア（Sentiment/Confidence/Intensity/Expectation Gap/Timeliness）を出力する。8軸スコアへの変換は以下のマッピングで行う:

| 8軸 | ISQソース | 変換ルール |
|-----|----------|----------|
| 割安度 | Phase 1（ファンダ）のSentiment + DCF乖離率 | Sentiment(-1〜+1)を0〜100に線形変換、DCF乖離率で加減点 |
| 稼ぐ力 | Phase 1 の財務データ（ROE・営業利益率・EBITDAマージン等）から直接算出 | ISQ経由せず生データから算出 |
| 成長性 | Phase 1 の財務データ（成長率・PEG・設備投資比率等）から直接算出 | ISQ経由せず生データから算出 |
| 安全性 | Phase 2（信用リスク）のSentiment + 自己資本比率 | Sentiment を0〜100に変換、自己資本比率で補正 |
| トレンド | Phase 3（テクニカル）のSentiment | Sentiment を0〜100に線形変換 |
| 需給 | Phase 4（需給）のSentiment | Sentiment を0〜100に線形変換 |
| 還元力 | Phase 1 の配当分析データから直接算出 | ISQ経由せず生データから算出 |
| 事業独占力 | Phase 1 の財務データ（10年分のoperatingMargin, roe, fcfMargin, sgaRatio）から直接算出 | ISQ経由せず生データから算出 |

**Sentiment → 0〜100 変換式**: `score = (sentiment + 1) * 50`（-1→0、0→50、+1→100）

稼ぐ力・成長性・還元力・事業独占力はISQスコアではなく、財務データの生値から直接算出する。ISQは方向性を示す指標であり、これらの軸は具体的な財務指標のほうが正確にスコア化できるため。

## データソース

- **ラジ株ナビMCP v2**（2026-04-04より）: 80指標が利用可能。財務データ（ROE・営業利益率・EBITDAマージン・FCFマージン・SGA比率・設備投資比率・DOE等）をEDINET経由で最大10年分取得できる。
- **JQuants API v2**: 株価データ（日足OHLCV）。Freeプランは直近12週分。
- **TDnet MCP**: 適時開示情報。APIキー不要。

## 基準値について

全軸の基準値は50固定。業界平均の実データは使用しない。スコアは絶対基準のみで算出する。レーダーチャートのグレー破線（基準値50）は「普通」の水準を示す参考線。

## JQuants無料プランでの制約

| 項目 | 制約 | 対応 |
|------|------|------|
| 株価データ | 直近12週のみ | 株価チャートは12週分で表示 |
| SMA200 | 200日分のデータが不足 | SMA50のみ表示、注記を添える |
| SMA200表示 | Light以上で可能 | `.env` の `JQUANTS_PLAN` の値で判定 |

### JQUANTS_PLANの値とSMA200表示

| JQUANTS_PLAN | SMA200 | データ期間 |
|-------------|--------|----------|
| `free`（デフォルト） | 非表示 | 直近12週 |
| `light` | 表示 | 2年分 |
| `standard` | 表示 | 5年分 |
| `premium` | 表示 | 全期間 |

SMA200非表示の場合: HTMLテンプレートでSMA200の系列（`datasets`）自体を描画しない。凡例からも除外する。`priceData[].sma200` が `undefined` の場合にこの挙動となる。

## 比較対象の提供（ガイド準拠）

全ての数値に対して以下の比較軸を提供する:

1. **絶対基準との比較**: 「PER 7.3倍は一般的に割安水準（8倍以下）」
2. **過去の自分との比較**（縦の比較）: 「PBR 1.2倍（前年 1.4倍 → 改善）」

これにより、数値が良いのか悪いのかを初心者でも判断できる。

- コントラスト比3:1以上
- 凡例はグラフ近傍に配置
- 白黒印刷でも判別可能な設計を意識
