# PDF分析レポート機能 設計書

## 概要

kabu-dexterの総合分析結果を、初心者にもわかりやすいビジュアルPDFレポートとして出力する機能。レーダーチャートと横棒グラフで7軸のスコアを可視化し、数値の意味づけ（業界比較+過去比較）をストーリー型で伝える。

## 背景と目的

### 課題
- 現在の総合分析出力は専門用語が多く、初心者には理解が困難
- 数値が並ぶだけで「結局どうすればいいのか」がわからない
- 情報量が多すぎて圧倒される

### 目的
- 投資初心者〜中級者が一目で銘柄の全体像を把握できるレポート
- 数値を「どう見ているか」（業界比較+過去比較）を常に添えて解釈を提供
- 保存・共有しやすいPDF形式で出力

### 将来展望
- ラジ株ナビ（Web版）への組み込み
- note有料プランでのレポート提供

## ターゲットユーザー

- A) 株式投資自体が初めて（PERやROEに馴染みがない）
- B) 投資経験はあるが分析ツールは使ったことがない
- C) 分析手法は知っているがツールの操作がハードル

全レベルをカバーする。

## 技術スタック

- **テンプレート**: HTML + Chart.js（CDN）
- **PDF化**: Puppeteer（ヘッドレスChrome）
- **ランタイム**: Bun（既存プロジェクトに合わせる）
- **理由**: HTML→PDF変換はデザイン自由度が高く、ユーザー環境にPythonを要求しない

## デザイン方針

### デジタル庁ダッシュボードガイド準拠

参考: https://www.digital.go.jp/resources/dashboard-guidebook

以下の原則に従う:

#### 2大原則
1. **「知りたいことを知れる」** - シンプル、意味のある順列、強弱をつける
2. **「誤解を生まない」** - わかりやすく表記、データを定義、表現を歪曲しない、メタ情報を記載

#### カラーパレット

| トークン | HEX | 用途 |
|---------|-----|------|
| Blue 600 | `#3460FB` | キーカラー、業界平均以上のスコア |
| Blue 900 | `#0017C1` | ヘッダーテキスト |
| Blue 400 | `#7096F8` | SMA50ライン |
| Blue 200 | `#C5D7FB` | 出来高、薄い背景 |
| Blue 50 | `#D9E6FF` | セクションタイトル背景 |
| Red 600 | `#FE3939` | 業界平均以下のスコア |
| Solid Gray 400 | `#999999` | 業界平均ライン |
| Solid Gray 200 | `#CCCCCC` | ボーダー |
| Standard BG | `#F8F8FB` | ページ背景 |

#### グラフルール
- グラフの色数は1〜5色に絞る
- コントラスト比3:1以上確保
- 棒グラフの原点は必ず0
- データ点にマーカーを付ける
- 凡例はグラフの近傍に配置
- 色のみで分類を識別しない（形状でも区別）
- タイトルにデータ種別を明記
- メタ情報（データソース・更新日）を記載

#### レイアウト
- 左上→右下の視線の流れ
- 全体→詳細の階層構造
- グリッドベース

#### レーダーチャートについて
デジタル庁ガイドの推奨グラフ（折れ線・棒・面・円）にレーダーチャートは含まれない。本レポートでは直感的な理解のためにレーダーチャートを採用しつつ、正確な比較用に横棒グラフも併記する。カラーパレット・レイアウト原則・アクセシビリティはガイドに準拠する。

## レポート構成

1つの連続したレポート（強制改ページなし）。PDF化時に自然な位置で改ページされる。

```
┌─────────────────────────────────────────────┐
│ ヘッダー: 銘柄情報                            │
├──────────────────────┬──────────────────────┤
│ レーダーチャート       │ 総合コメント          │
│ （7軸、自社+業界平均） │ （ストーリー型）      │
├──────────────────────┴──────────────────────┤
│ 7軸スコアカード（横一列）                      │
├─────────────── 区切り線 ────────────────────┤
│ 各軸の詳細（カード型、2カラムグリッド）         │
│ + 横棒グラフ（業界平均との差分）               │
├─────────────── 区切り線 ────────────────────┤
│ 株価チャート（折れ線+出来高+SMA）              │
├─────────────────────────────────────────────┤
│ リスク要因                                   │
├─────────────────────────────────────────────┤
│ データソース・分析手法補足                     │
├─────────────────────────────────────────────┤
│ フッター（免責事項・更新日）                   │
└─────────────────────────────────────────────┘
```

### ヘッダー
- 銘柄名（大きく）、銘柄コード、セクター
- 現在株価、時価総額、分析日

### レーダーチャート
- 7軸、0〜100スケール
- 自社スコア: Blue 600（半透明塗り、丸マーカー）
- 業界平均: Solid Gray 400（破線、三角マーカー）
- 凡例はチャート近傍に配置

### 総合コメント
- ストーリー型で3〜5行
- 「この会社は〜が強いが、〜に陰りがあり、〜」の形式
- 数値を根拠として織り込む

### 7軸スコアカード
- 横一列のカード型
- 各カード: 軸名 + スコア数値 + ミニバー
- 業界平均以上→Blue、以下→Red で色分け

### 各軸の詳細
- 2カラムグリッドでカード表示
- 各カード: 軸名 + スコア(/100) + プログレスバー（業界平均マーカー付き）+ 根拠テキスト
- 根拠は「数値 → 比較対象 → 解釈」の3点セット
  - 業界平均との比較（横の比較）
  - 過去の自分との比較（縦の比較）
- 横棒グラフ: 7軸を並べて業界平均との差分を可視化（正=Blue/負=Red、原点0）

### 株価チャート
- 直近12週の日足（JQuants無料プランの範囲）
- 折れ線: 終値（Blue 600、マーカー付き）
- 破線: SMA50（Blue 400）
- 棒グラフ: 出来高（Blue 200、第2軸）
- SMA200: 無料プランでは算出不可。Light以上で表示（注記付き）

### リスク要因
- 箇条書き2〜3項目
- 分析結果から特に懸念される点を抽出

### フッター
- データソース: ラジ株ナビ / JQuants / TDnet
- 更新日
- 免責事項（投資助言ではない旨）

## 7軸スコア定義

全軸0〜100。50を「普通（業界平均水準）」とする。

### 1. 割安度（Valuation）
| 元データ | 変換ルール |
|---------|----------|
| DCF乖離率 | +30%以上割安→90、±0%→50、-30%以上割高→10 |
| PER（vs業界平均） | 業界平均より低い→加点 |
| PBR | 1.0以下→加点 |
| ネットネット判定 | 該当→大幅加点 |

### 2. 稼ぐ力（Profitability）
| 元データ | 変換ルール |
|---------|----------|
| ROE | 15%以上→80〜、10〜15%→60〜、10%未満→40以下 |
| 営業利益率 | 業界平均比で加減点 |
| ROA | 補助指標として加点 |

### 3. 成長性（Growth）
| 元データ | 変換ルール |
|---------|----------|
| 売上成長率（3年CAGR） | 10%以上→80〜、5〜10%→60〜 |
| EPS成長率 | 同上の基準 |
| PEGレシオ | 1.0未満→加点 |

### 4. 安全性（Safety）
| 元データ | 変換ルール |
|---------|----------|
| Altman Z'' | 安全圏→80〜、グレー→50〜、危険→20以下 |
| 自己資本比率 | 50%以上→加点 |
| 金融業 | Z''スキップ、自己資本比率中心で判定 |

### 5. トレンド（Trend）
| 元データ | 変換ルール |
|---------|----------|
| SEPAステージ | S2→80〜、S1→60〜、S3→40〜、S4→20〜 |
| ダウ理論 | 上昇→加点、下降→減点 |
| グランビル | 買いシグナル→加点、売りシグナル→減点 |

### 6. 需給（Supply-Demand）
| 元データ | 変換ルール |
|---------|----------|
| 信用倍率 | 1〜3倍→80〜、3〜5倍→60〜、5倍超→40以下 |
| 出来高トレンド | 増加→加点 |
| モンテカルロ上昇確率 | 60%以上→加点 |

### 7. 還元力（Shareholder Return）
| 元データ | 変換ルール |
|---------|----------|
| 配当利回り | 3%以上→加点 |
| 増配率トレンド（3〜5年） | 加速→加点、鈍化→減点（花王パターン検出） |
| 配当性向 | 30〜50%→健全で加点、80%超→無理で減点 |
| DOE | 3%以上→加点 |
| 自社株買い実績 | あり→加点 |
| 総還元性向 | 50〜80%→加点 |
| 無配銘柄 | 自社株買いのみで評価、なければ20以下 |

## ファイル構成

```
src/skills/pdf-report/
├── SKILL.md              # スキル定義（トリガー条件・入力仕様）
├── templates/
│   ├── report.html       # HTMLテンプレート（本番用）
│   └── mockup.html       # デザインモックアップ（参考）
├── generate.ts           # メイン生成ロジック
│                          - 分析結果JSON → HTMLレンダリング → PDF出力
└── scores.ts             # 7軸スコア変換ロジック
                           - 各分析フェーズの結果を0〜100に正規化
```

## 統合フロー

```
comprehensive-analysis（総合分析）
  └─ Phase 7 完了後
     └─ 「PDFレポートを出力しますか？」と確認
        └─ Yes → pdf-report スキルを呼び出し
           ├─ 分析結果を7軸スコアに変換（scores.ts）
           ├─ HTMLテンプレートにデータ注入（generate.ts）
           ├─ Puppeteer でPDF化
           └─ output/{code}_report_{date}.pdf に保存
```

- 総合分析の出力は従来通り（ターミナルにテキスト）
- PDFは追加オプション。分析完了後にユーザーの意思で生成

## JQuants無料プランでの制約

| 項目 | 制約 | 対応 |
|------|------|------|
| 株価データ | 直近12週のみ | 株価チャートは12週分で表示 |
| SMA200 | 200日分のデータが不足 | SMA50のみ表示、注記を添える |
| SMA200表示 | Light以上で可能 | プラン検出して表示/非表示を切り替え |

## 比較対象の提供（ガイド準拠）

全ての数値に対して以下の比較軸を提供する:

1. **業界平均との比較**（横の比較）: 「PER 15倍（業界平均20倍 → 割安）」
2. **過去の自分との比較**（縦の比較）: 「PBR 1.2倍（前年 1.4倍 → 改善）」

これにより、数値が良いのか悪いのかを初心者でも判断できる。

## アクセシビリティ（ガイド準拠）

- グラフの系列は色+形状（circle/triangle/rect）で区別
- コントラスト比3:1以上
- 凡例はグラフ近傍に配置
- 白黒印刷でも判別可能な設計を意識
