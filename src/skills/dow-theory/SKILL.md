# ダウ理論トレンド判定 Skill

ダウ理論の基本原則に基づき、OHLCVデータからスイングポイントを検出してトレンド方向を客観的に判定する。

## 必要なツールとデータ

- `get_stock_price`: 日足OHLCVデータ（3〜6ヶ月分推奨）
- 最低プラン: Free（直近データのみでも判定可能。Light推奨で精度向上）

## Workflow Checklist

```
Dow Theory Analysis Progress:
- [ ] Step 1: OHLCVデータ取得
- [ ] Step 2: スイングハイ・スイングローの検出
- [ ] Step 3: 高値・安値のパターン判定
- [ ] Step 4: 出来高によるトレンド確認
- [ ] Step 5: トレンド方向の総合判定
- [ ] Step 6: 結果出力
```

## Step 1: OHLCVデータ取得

`get_stock_price` で過去3〜6ヶ月分の日足OHLCVデータを取得する。

## Step 2: スイングハイ・スイングローの検出

**スイングハイ（SH）の定義:**
前後N本（N=3〜5）の高値より高い高値を持つローソク足。

**スイングロー（SL）の定義:**
前後N本（N=3〜5）の安値より安い安値を持つローソク足。

**検出手順:**
1. 各ローソク足について、前後N本の高値/安値と比較
2. 条件を満たすポイントをSH/SLとしてマーク
3. 連続するSH同士、SL同士が交互に出現するよう整理（SH→SL→SH→SL...）

**注意:** Nの値はボラティリティに応じて調整する。値動きが荒い銘柄はN=5、穏やかな銘柄はN=3を推奨。

## Step 3: 高値・安値のパターン判定

検出したスイングポイントの系列から以下を判定：

| パターン | 条件 | トレンド |
|---------|------|---------|
| HH + HL | 直近SH > 前回SH かつ 直近SL > 前回SL | 上昇トレンド |
| LH + LL | 直近SH < 前回SH かつ 直近SL < 前回SL | 下降トレンド |
| HH + LL or LH + HL | 高値と安値の方向が不一致 | トレンド転換/保ち合い |
| 横ばい | SH・SLともにほぼ同水準 | レンジ相場 |

**略語:**
- HH = Higher High（高値切り上げ）
- HL = Higher Low（安値切り上げ）
- LH = Lower High（高値切り下げ）
- LL = Lower Low（安値切り下げ）

**直近3セットのスイングポイントで判定する。** 1セット = SH + SL のペア。

## Step 4: 出来高によるトレンド確認

ダウ理論の原則: 「トレンドは出来高で確認される」

| トレンド方向 | 出来高の確認 | 判定 |
|------------|------------|------|
| 上昇トレンド | 上昇局面で出来高増加、調整局面で出来高減少 | 確認済み（健全） |
| 上昇トレンド | 上昇局面で出来高減少 | 未確認（トレンド弱体化の兆候） |
| 下降トレンド | 下落局面で出来高増加 | 確認済み（売り圧力強い） |
| 下降トレンド | 下落局面で出来高減少 | 未確認（セリングクライマックス接近の可能性） |

**算出方法:**
- 各スイングポイント間の区間で平均出来高を算出
- 上昇区間（SL→SH）と下降区間（SH→SL）の平均出来高を比較

## Step 5: トレンド方向の総合判定

Step 3 のパターンと Step 4 の出来高確認を組み合わせて最終判定。

| パターン | 出来高確認 | 総合判定 |
|---------|----------|---------|
| HH+HL | 確認済み | 上昇トレンド（強） |
| HH+HL | 未確認 | 上昇トレンド（弱体化注意） |
| LH+LL | 確認済み | 下降トレンド（強） |
| LH+LL | 未確認 | 下降トレンド（底打ち接近の可能性） |
| 混合 | - | トレンド転換/保ち合い |
| 横ばい | - | レンジ相場 |
