# 決算カレンダー Skill

決算日の確認と決算リスクの事前把握を行う情報提供スキル。ISQスコアは算出しない（情報整理のみ）。

## 必要なツールとデータ

- `get_financials`: `fins_announcement` エンドポイントで決算発表日を取得
- Web検索: 決算日が API で取得できない場合のフォールバック
- 最低プラン: Free

## Workflow Checklist

```
Earnings Calendar Progress:
- [ ] Step 1: 決算発表日データ取得
- [ ] Step 2: 次回決算日の特定と日数計算
- [ ] Step 3: 過去の決算サプライズ確認
- [ ] Step 4: 結果出力
```

## Step 1: 決算発表日データ取得

`get_financials` で決算発表スケジュールを取得する。

**取得する情報:**
- 次回決算発表予定日
- 決算期（1Q / 2Q / 3Q / 本決算）
- 過去4〜8四半期の決算発表日と実績

**フォールバック:** API で取得できない場合は、Web検索で「{銘柄名} 決算発表日」を検索。JPX適時開示や会社IRページで確認。

## Step 2: 次回決算日の特定と日数計算

```
決算までの日数 = 次回決算発表日 - 今日の日付
```

**警告基準:**

| 日数 | 警告レベル |
|------|----------|
| 14日以内 | 決算直前警告（ボラティリティ大幅上昇リスク） |
| 14〜30日 | 決算接近通知 |
| 30日超 | 通常（特別な警告なし） |

## Step 3: 過去の決算サプライズ確認

過去4〜8四半期の決算について、以下を確認：

- **コンセンサス予想 vs 実績**: 上振れ（Beat）/ 下振れ（Miss）/ 一致（Meet）
- **決算発表翌日の株価反応**: 上昇/下落幅（%）
- **上方修正/下方修正の履歴**

**データソース:**
- `get_financials` で取得可能な範囲
- 不足分は Web 検索で「{銘柄名} 決算 サプライズ」等

**注意:** コンセンサス予想が取得できない場合は「予想データ取得不可」と明記し、会社予想 vs 実績の比較で代替する。
